# [THE FOUNDATION]
- **Core Stack**: Go (v1.22) + Wails (v2.8.0) + React (Vite/TS).
- **Styling**: TailwindCSS.
- **State Management**: Zustand (Frontend), Go Struct/App Bindings (Backend).
- **Key Dependencies**:
    - `github.com/anacrolix/torrent`: Real-time torrent streaming and management.
    - `mpv`: External media player control (via `internal/mpv`).
    - `sqlite3`: Local library and metadata storage.
    - `OpenSubtitles API`: Remote subtitle fetching.
- **Routing**: Likely handled within React (SPA).
- **Global State Topology**: Zustand stores for torrents, playback, and navigation.

# [THE FORGE]
- **Feature**: Video Playback Engine.
- **Current Activity**: Diagnosed and fixed the MPV engine silently failing to launch on Windows due to an invalid Unix Domain Socket path. Shifted the architecture to natively use `\\.\pipe\media-maestro-mpv` named pipes via `natefinch/npipe.v2`.
- **Files Under Mutation**: `internal/mpv/client.go`, `internal/mpv/manager.go`.

# [THE GRAVEYARD]
- **Duplicate Logic**: `engine.go` and `stream.go` were exact duplicates, causing re-declaration errors.
- **Misplaced Imports**: Multiple files had `import` statements at the bottom, breaking Go's grammar.
- **Type Mismatch**: `mac.TitleBar` required a pointer in `main.go`.
- **API Changes**: `anacrolix/torrent` f.Path is not a string; `f.DisplayPath(info)` is required. `ActivePeers` replaces `ConnectedPeers`.
- **Duplicate Utility**: `findMPVBinary` was duplicated in `app.go` and `internal/mpv/manager.go`. Consolidated in `manager.go` and improved search paths.
- **Frontend Type Mismatch**: `SubtitleDisplay.tsx` was incorrectly treating the Wails `unlisten` function as a Promise. Fixed to call it directly.
- **Wails API Casing**: `torrent-store.ts` had namespace issues for `TorrentInfo` and missing `await` for `TorrentGetStreamURL`. Consolidated with `torrent` namespace from Wails models.
- **Missing Build Assets**: `index.html`, `main.tsx`, `App.tsx`, and Tailwind/PostCSS configs were missing from the frontend directory, causing build failures. Restored with default premium configurations.
- **CGO Dependency**: `github.com/mattn/go-sqlite3` required CGO which failed at runtime on Windows without a C compiler. Switched to pure-Go `modernc.org/sqlite`.
- **File Picker Missing**: Original UI mocks shifted states instead of launching a file picker. Restrung `MediaGrid` to dynamically invoke `App.OpenFilePicker()`.
- **Dead Player Component**: `AppShell` imported placeholder `SeanimePlayer.tsx` instead of the real `MaestroPlayer.tsx` with full controls/seek/subtitles.
- **Missing CSS Tokens**: `surface-4`, `text-tertiary`, `accent-hover` never defined; `glass-heavy` class, `animate-slide-up/fade-in/scale-in` keyframes missing. All added.
- **Dead Import**: `persist` import left in `ui-store.ts` after prior refactor. Removed.
- **Duplicate Switch Cases**: `use-keyboard.ts` had duplicate `case '1'`-`'3'` entries (number-seek vs Alt+mode-switch). Alt guard unified.
- **Type Any**: `VolumeSlider.tsx` used `(s: any)` selectors. Fixed to proper typed selectors.

# [THE BLOCKERS]
- **None**: All identified issues resolved. App boots, file picker works, real player loads with controls.
